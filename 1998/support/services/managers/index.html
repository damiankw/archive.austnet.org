Network and Channel Policies

1.1 What AustNet policies do I need to be aware of?

    • No clones
    • No Abusive behaviour
    • No Flooding
    • No Mass messaging/advertising/inviting
    If you or any member of your channel is found to be doing any of these things you face suspension of your channel and possible ejection from AustNet completely. It is tempting for new managers to mass advertise their channels but such behaviour is greatly annoying to users and is not permitted.

1.2 What channel policies can I make?

    Your channel is free to make and enforce any rules it chooses, provided that its activities do not conflict with the AustNet policies. Channel operators aren't required to have any reason for banning users, and server admins will not interfere with the normal business of your channel.

Channel Settings and Bot Control

2.1 How do I alter channel settings?

    Use the command /msg ChanOP set #channel setting value, where setting is one of:
    • restrict level require certain access to join channel
    • keeptopic level set the level of users who may change the topic
    • url URL display a URL or short message to people who join
    • mode level modes set and lock modes on the channel
    • oprestrict on|off only channel members may hold ops
    • mustid on|off users must identify for ops/protection
    • telldel on|off owner gets note when someone uses deluser
    • nonote on|off only the manager may send notes to the channel

2.2 How do I drop a channel?

    The command /msg ChanOP drop #channel will cause ChanOP to deregister the channel. If you wish to transfer ownership to another person you should instead see an ASD admin who can transfer the channel without deleting its database.

2.3 How do I make the bot join and part?

    /msg ChanOP join #channel and /msg ChanOP part #channel make ChanOP join and part. All commands work as normal even if ChanOP is not on channel, but you cannot use `command-style in-channel instructions. Do not abuse these commands.

2.4 How do I make the bot speak?

    The commands /msg ChanOP speak #channel and /msg ChanOP act #channel have been disabled due to user abuse. There are no plans for them to be re-enabled.

Channel Recovery

3.1 How do I clear the modes set in my channel?

    If you find yourself locked out of your own channel use /msg ChanOP clean #channel to clear modes s, t, i, n, k, p, l and m. For modes o and b see below.

3.2 How do I deop all the ops in my channel?

    The command /msg ChanOP mdeop #channel will deop all the ops lower than your level. Do not abuse this command.

3.3 How do I clear all the bans set in my channel?

    The instruction /msg ChanOP unban #channel all will clear all bans set lower than your level.

User Management

4.1 How do I add users to my channel?

    Use the command /msg ChanOP adduser #channel nick level to add the user and set auto-op on. If you do not specify a level the user is added at level 100. The level at which you set a user determines what commands he may use; to see a list of commands and their levels use /msg ChanOP commands #channel. Users of levels above 150 can add users themselves so issue this authority with care. /msg ChanOP help levels also provides a useful reference for access levels.

    All users must be registered with NickOP before they can be added to channel databases.

4.2 How do I change a user's settings?

    The command /msg ChanOP setuser #channel nick setting value alters user settings, where setting is one of:
    • level newlevel modify user's access level
    • aop on|off set user +o on join
    • aov on|off set user +v on join (only if AOP is not set)
    • protect on|off protect the user from kicks and deops
    • suspend off|time in minutes will suspend a user's access for the specified time period

4.3 How do I delete users?

    Use the command /msg ChanOP deluser #channel nick to delete users.
