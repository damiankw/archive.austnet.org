This guide is to ease your migrations from dal.net to Austnet and give you information about the basics of our services.

AustNet Rules
Upon connecting to AustNet, you agree to our Terms of service. It is advised that you read this before proceeding as this outlines many of the rules on Austnet, some of which differ from dal.net.
Separate to this, each server also has its own set of rules for its users, which can be found by /motd when connected to the network.

Services
Formerly you had access to several services. These were chanserv, nickserv and memoserv, which allowed channel registration, nickname registration and sending memo's respectively. On AustNet these facilities (and more) are available, however the services have different names. We have ChanOP, NickOP, NoteOP and our very own LoveOP, ChanOP handles channel registration, much like chanserv. NickOP is provided for nickname registration and NoteOp for sending memo/notes to other registered users. LoveOP is for people who wish to record their online romances or get net married for fun.
Each of AustNet's user services have their own help files. These can be accessed by
/msg servicename help
ChanOP: /msg chanop help
NickOP: /msg nickop help
NoteOP: /msg noteop help
LoveOP: /msg loveop help

Nickname Registration
Before you register a nickname you need to find one that is not already taken, bearing in mind austnet has a limit of 15 characters for nicknames.
To check if a nickname is already registered type:
/msg nickop info nickname
where nickname is the nickname you wish to register.
If the nickname is not registered then you may register it by changing to it then typing
/msg nickop register yourpassword youremail
yourpassword should be a password between 4 and 25 characters long and must contain atleast one non alphabetical character. Passwords are also case sensitive.
youremail should be a valid email address of yours. This address is only used to email your password in the event that you forget it.

Example:
/msg nickop@austnet.org register V4d3r chips@are.yum.com
In this example your password would be V4d3r and email chips@are.yum.com

Identifying to services?
Each time that you connect to Austnet you will need to identify to NickOP. This can be done by typing
/msg nickop@austnet.org id password
Example
/msg nickop@austnet.org id V4d3r

Nickname Protection
Austnet advises that you enable nickname protection on your nick so that others cannot use it for more than 120 seconds. This can be done by typing
/msg nickop set kill on
For more information on NickOP type /msg nickop help or join #asd

Channel Registration
Austnet does not require that you wait lengthy periods of time to register a channel -- it can be done instantly. First you need to think of a channel name and ensure its not registered. You can check if its registered by typing
/msg chanop info #channel
If you find a name you like that is not registered join the channel and type
/msg chanop register #channel
Once you have done this chanop should message/notice you asking you to agree to the terms and conditions of channel ownership, read these terms then agree.
Presto you are now a channel owner on Austnet.

Channel Protection
It is advised when creating a channel that you set the following options
/msg chanop set #chan mode 190 +tn-psklmi
This locks the channel modes to 190
/msg chanop set #chan mustid on
This will require users to be identified to use access
/msg chanop set #chan oprestrict on
This will require users to have access in the channel to be opped.
These settings are suggested to ensure unauthorised people can not get ops.


For more help, you can ask in #help, or contact help@austnet.org. For services help, contact ASD helpers in #asd, or email services@austnet.org.


Unlike dalnet where commands need to be sent to the full service name chanserv@... on austnet you can send them to ChanOP. The exception is that when you use a password you should use nickop@austnet.org.
On AustNet, you do not need to identify to chanserv to be considered the owner of a channel. Once you identify to nickop, you are recognised as the owner of that nickname and the channel it owns, by services.
AustNet does not have a /nickop or /chanop command, however we do recommend making an alias to use these.
For mIRC users:
/msg nickop@austnet.org $1- and
/msg chanop $1-
 Nicknames will expire after 21 days of not identifying.
Channels will expire after 14 days of no use by the ops.
If you get an IRC trojan/virus visit the AustNet Virus Assistance Channel #ava
