Server applications to connect to the austnet IRC network are submitted for approval and are then voted in or rejected by the server admin of the austnet.

Applications are only accepted via submission of the application form to routing@austnet.org. This page is designed to help future applicants in ensuring they have the correct technical facilities, bandwidth and qualifications.

The application form can be downloaded here.

It is mandatory that applicants complete and submit the form to routing@austnet.org Server applications in any other method are simply ignored. All fields within the application form are required to be filled to the best of your ability and with all due honesty.

Please Note: Submission of the application is reviewed extremely pedanticly. Failure to submit the form with all honesty and correct details may result in disallow of application.

  1. The IRCD must be able to run 24 hours a day, 7 days a week.

  2. It is strongly suggested that you meet the following requirements:

    1. You should have at least a P100 class machine with 32MB minimum OR a P60 class machine with 64MB minimum. This must be physical RAM, not VM.

    2. You should be running some flavour of UNIX (Linux, BSD, etc.)

    3. If you are applying from the US or Canada your connection to the Internet should be at least a T1. If you are applying overseas, it should be at least a 256k connection.

  3. This server must be available for the long term. If you intend on taking this server down in the "near future", please do not apply. You can use your best judgement on our definition of "near future". We're looking for servers that will be reliable.

  4. You must be fully aware and have read the charter, as well as any other rules and regulations that may be ammended from time to time.

  5. You hold harmless all those involved in the network, including but not limited to its server admins, hub admins and users.

  6. 100 data packets (56 data bytes) is required on each ping

The austnet IRC daemon is not a publicy available source code.
When, or if, your server is voted in, you will receive the source code.

All enquiries or suggestions should be forwarded to routing@austnet.org
