Server Patches

Patch Name Description
services.modnick
(Version 1.4)		 This patch forces nickname changes caused by NickOP when a user has failed to identify on a registered nickname, consequently not disconnecting the user, rather enforcing a nickname change instead. This patch has been under construction since May 1997 and will be online July 1997.
server.burst
(Version 1.4)		 AustNet servers now use a compression method called "tokenisation", a proceedure that reduces the data sent during a server's resynch burst by aliasing certain commands and information. Similarly, AustNet servers run at P11, a protocol that sends only necessary information to other servers, effectively compressing the data traffic around the network. Other traffic, such as client quits on a disconnecting server, are computed via internal algorithms.
server.localhelp
(Version 6.2)		 All AustNet servers have the service's help pages locally, essentially making your commands to services regarding assistance much faster. Each server comes with it's own internal help library leading to a faster response for help files. Also, all servers have an upgraded /quote HELP command that lists the parameters to each server command.
client.list
(Revision 1.2)		 The /list command generated by clients is meant to return all of the current channels on the network, however, we have found that ChanOP residing thousands of channels has caused new users to be often lost and unable to locate a popular channel. Consequently, AustNet has implemented a patch that only lists channels to new users when there is more than one user in any specific channel that ChanOP resides on. This means that the /list will only return populated channels, instead of empty and vacant channels. The user mode "+l" can be set for the client to override this list patch.
server.whomask
(Revision 2.3)		 This patch effectively stops "spammers" from mass-advertising on the network by not returning wide range domain searches. Consequently, abusive users are not able to mass-advertise and send unsolicited messages to AustNet users. This patch was generated January 1997 and has been online since March 1997.
oper.umode
(Revision 1.1)		 An IRC Operator's user modes have expanded to include a seperate mode for reporting client connectivity, server management notices and server generated alerts. This effectively allows some IRC Operators to ignore certain aspects of server management to focus on other tasks.
services.netmode
(Revision 1.3)		 On AustNet, the services are able to issue themselves Channel Operator access on any channel they wish, effectively allowing them to maintain the network. These powers allow services to be more potent than any other "bot" on the network, and ensure channel security and integrity. Services are consequently able to change any channel mode on or off the channel irrespective.
services.umode
(Revision 5.2)		 AustNet services are not able to be kicked, banned or deopped from any channel on the network, allowing them to recover channels that possibly could of have been taken over during a network resynch.
server.bans
(Revision 3.4)		 All AustNet servers listen to central services and implement network wide bans on request. This allows IRC Operators the ability to forcibly remove clients from the network by banning them from every possible server. If a server reconnects at a later date, after the ban has been set, during the resynch period, the network bans will be sent downstream, consequently all servers will not allow that particular hostmask access to the network. If, for some reason, a client has connected to the network who is on the services ban list, the network ban will be placed once again to ensure security.
